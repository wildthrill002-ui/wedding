# Свадебный сайт — Артём & Анна

Структура проекта:

```
wedding-site/
├── index.html
├── css/style.css
├── js/
│   ├── content.js        ← ВСЕ тексты сайта (даты, адрес, таймлайн, подписи формы)
│   └── main.js            ← рендер, анимации (GSAP/Lenis), отправка RSVP
├── images/                 ← фото (сейчас — 3 плейсхолдера с котом Симбой)
├── google-apps-script/
│   └── Code.gs             ← код для Google Apps Script (приём формы → Google Таблица)
└── README.md
```

## 1. Как посмотреть локально

Просто открой `index.html` в браузере — двойным кликом или через
`Live Server` в VS Code. Отдельный сервер не нужен.

## 2. Как поменять тексты, дату, адрес

Всё редактируется в одном файле — **`js/content.js`**. HTML трогать не
нужно. Там же меняются пункты таймлайна (добавить/убрать строку — просто
добавь/убери объект в массиве `timeline.items`) и варианты алкоголя.

## 3. Как заменить фото

Фото хранятся в `images/`. Чтобы поменять или добавить фото в галерею —
положи файл в `images/` и пропиши его имя в `js/content.js` →
`gallery.images`. Фото в hero (начало сайта) и в футере меняются так же —
просто замени файл и, если нужно, путь в `js/content.js` / `index.html`.

## 4. Подключение Google Sheets (приём ответов формы)

Ответы формы отправляются на **Google Apps Script Web App**, который
дописывает строку в Google Таблицу. Настройка — один раз, 5–10 минут.

### Шаг 1 — создай Google Таблицу
1. Открой [sheets.google.com](https://sheets.google.com) → **Новая таблица**.
2. Назови её как удобно, например «RSVP — свадьба».

### Шаг 2 — вставь скрипт
1. В таблице: **Расширения → Apps Script**.
2. Открой файл `google-apps-script/Code.gs` из этого проекта, скопируй
   весь код и вставь в редактор Apps Script (замени содержимое
   `Code.gs` там).
3. Сохрани (значок дискеты или `Ctrl+S`).

Скрипт сам создаст в таблице лист с именем `RSVP` и заголовки столбцов
при первой отправке формы.

### Шаг 3 — разверни как Web App
1. В Apps Script нажми **Deploy → New deployment**.
2. Тип — **Web app**.
3. Настройки:
   - **Execute as**: `Me` (твой аккаунт)
   - **Who has access**: `Anyone` (иначе форма гостей не сработает)
4. Нажми **Deploy**. Первый раз попросит авторизацию — разреши доступ
   своему аккаунту (это твой собственный скрипт, доступ дают один раз).
5. Скопируй появившийся **Web app URL** — выглядит так:
   `https://script.google.com/macros/s/AKfycb.../exec`

### Шаг 4 — вставь URL в сайт
Открой `js/main.js`, в самом верху:

```js
const GAS_ENDPOINT = "ВАШ_WEB_APP_URL";
```

Замени `"ВАШ_WEB_APP_URL"` на скопированную ссылку. Сохрани файл.

Теперь при отправке формы каждая заявка гостя будет новой строкой в
таблице `RSVP` со столбцами: Дата отправки, ФИО, Телефон, Соцсети,
Присутствие, Количество гостей, Трансфер туда, Трансфер обратно,
Коттедж, Алкоголь, Еда, Пожелание.

**Важно:** Apps Script Web App не отдаёт CORS-заголовки, поэтому сайт
отправляет данные в режиме `no-cors` и не может прочитать ответ сервера
напрямую — если запрос ушёл без сетевой ошибки, показываем экран
«Спасибо». Проверяй реальные заявки в самой таблице.

### Если что-то меняешь в Code.gs позже
Каждое изменение скрипта требует **нового деплоя**: Deploy → Manage
deployments → значок карандаша у существующего деплоя → **New version**
→ Deploy. URL при этом остаётся прежним.

## 5. Деплой на GitHub Pages

1. Создай новый репозиторий на GitHub (можно назвать `wedding` или
   `nasha-svadba`).
2. Загрузи в него всё содержимое папки `wedding-site/` (через
   `git init`, `git add .`, `git commit`, `git push`, или через
   веб-интерфейс GitHub — «Add file → Upload files»).
3. В репозитории: **Settings → Pages**.
4. **Source**: `Deploy from a branch`, ветка `main`, папка `/ (root)`.
5. Сохрани. Через минуту-две сайт будет доступен по адресу вида
   `https://твой-логин.github.io/название-репозитория/`.

## 6. Производительность

- Шрифты — только два, с `preconnect`.
- GSAP и Lenis подключены с `defer` и через `typeof` — если CDN
  недоступен, сайт всё равно работает (анимации откатываются на
  обычный CSS-reveal).
- Фото — с `loading="lazy"` (кроме первой фотографии в hero — она
  видна сразу, поэтому `eager`).
- Перед продакшеном сожми фото (TinyPNG/Squoosh) и, если добавляешь
  много своих фотографий в галерею, экспортируй их в WebP.
